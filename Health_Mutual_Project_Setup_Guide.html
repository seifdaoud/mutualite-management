Health Mutual Project - Setup Guide

1. Préparer l'Environnement

Prérequis :

  • Backend : JDK 17+, Maven 3.6+, MySQL/PostgreSQL
  • Frontend : Node.js 16+, Angular CLI
  • Docker (Facultatif) : Si vous utilisez Docker, vérifiez l'installation avec docker --version

2. Configurer le Backend

Étape 1 : Cloner ou Extraire le Projet

git clone <votre-repository-git>
cd health-mutual-project/backend

Étape 2 : Configurer la Base de Données

CREATE DATABASE mutualite;

Modifiez application.properties :

spring.datasource.url=jdbc:mysql://localhost:3306/mutualite
spring.datasource.username=root
spring.datasource.password=yourpassword

Étape 3 : Lancer le Backend

mvn spring-boot:run

Étape 4 : Tester les Endpoints

Utilisez Postman ou Swagger pour tester :

  • GET http://localhost:8080/api/utilisateurs
  • POST http://localhost:8080/api/contrats (avec un JSON de test)

3. Configurer le Frontend

Étape 1 : Naviguer vers le Dossier Frontend

cd ../frontend

Étape 2 : Installer les Dépendances

npm install

Étape 3 : Configurer l'API Backend

export const environment = {{
  production: false,
  apiUrl: 'http://localhost:8080/api',
}};

Étape 4 : Lancer le Frontend

ng serve

Accédez à http://localhost:4200

4. Tests

Backend

mvn test

Frontend

ng test

5. Optionnel : Conteneurisation avec Docker

Étape 1 : Configurer Docker Compose

version: "3.8"
services:
  backend:
    build: ./backend
    ports:
      - "8080:8080"
    environment:
      - SPRING_DATASOURCE_URL=jdbc:mysql://db:3306/mutualite
  frontend:
    build: ./frontend
    ports:
      - "4200:80"
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: mutualite

Étape 2 : Démarrer les Services

docker-compose up --build

6. Vérifications Finales

  • Backend : Vérifiez http://localhost:8080
  • Frontend : Vérifiez http://localhost:4200
